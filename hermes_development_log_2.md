## 実装した主な機能

### 1. 自然文操作
自然文から CLI コマンドへ変換する仕組みを実装した。

#### 例
- `今どこ？` → `pwd`
- `一覧見せて` → `ls`
- `差分見せて` → `git diff`
- `履歴見せて` → `git log --oneline -n 5`
- `前回のコマンドもう一回` → 前回コマンド再実行

---

### 2. 状態保持
Hermes 内部で状態を保持する仕組みを追加した。

#### 保存するもの
- `.hermes_cwd`  
  現在の作業ディレクトリ
- `.hermes_last_command`  
  前回コマンド
- `.hermes_last_command_cwd`  
  前回コマンド実行時のディレクトリ

#### 効果
- 自然文の連続操作でも文脈を保持できる
- `前回のコマンドもう一回` で、前回の場所も含めて再実行できる

---

### 3. 安全モード
危険な書き込み操作や破壊的操作を早期にブロックする仕組みを実装した。

#### ブロック対象
- 削除系  
  - `rm`
  - 「削除」「消して」「消す」
- Git 書き込み系  
  - `git add`
  - `git commit`
  - `git push`
  - `git merge`
  - `git rebase`
- ファイル変更系  
  - `mv`
  - `cp`
  - `chmod`
  - `chown`
  - `sed -i`
- 外部取得 / スクリプト実行系  
  - `curl`
  - `wget`
  - `bash`
  - `sh`

#### 返答例
```text
⚠️ 安全モードのため、削除系の操作は実行できません
💡 代わりに `ls` や `find` で対象を確認してください
```

```text
⚠️ 安全モードのため、Gitの書き込み操作は実行できません
💡 `git status`、`git diff`、`git log --oneline -n 5` は利用できます
```

---

### 自律モード v3
現在地が Git リポジトリ外でも、`ls` の結果から候補ディレクトリを探し、必要に応じて自動移動してから続きを実行するようにした。

#### 例
ホームディレクトリで `gitの状況を見て` を実行した場合:

1. `pwd`
2. `ls`
3. `hermes-agent` を候補として発見
4. 自律判断で `hermes-agent` に移動
5. `git status`
6. `git diff --stat`
7. `git log --oneline -n 5`

#### 出力例
```text
🤖 自律判断: /Users/hide/hermes-agent に移動して続行します
```

### 自律モード v6
候補ディレクトリを単純な優先順ではなく、スコアリングで比較して選択する方式に改善した。

#### 変更点
- 候補ごとに score を付与するようにした
- score の要素として以下を利用した
  - 依頼文のキーワード一致
  - 実在するディレクトリであること
  - Git リポジトリであること
- 最有力候補を選び、その score と理由一覧を表示するようにした

#### 例
- `claudeの状態を見て`
  - `everything-claude-code` を選択
  - score=14
  - 理由: `claude` キーワード一致、既存ディレクトリ、Git リポジトリ
- `hermesの状態を見て`
  - `hermes-agent` を選択
  - score=14
  - 理由: `hermes` キーワード一致、既存ディレクトリ、Git リポジトリ

#### 効果
これにより、Hermes Agent は単なる候補順選択ではなく、軽量な評価関数で対象を比較して選ぶ自律エージェントへ進化した。

### 自律モード v6.1
曖昧な依頼に対しては、`hermes-agent` を既定の作業対象として少し優先するように改善した。

#### 変更点
- `プロジェクト`
- `リポジトリ`

といった曖昧な依頼語が含まれる場合、`hermes-agent` に追加スコアを与えるようにした。

#### 例
- `このリポジトリを見て`
- `プロジェクトの状態を見て`

上記のような依頼では、`hermes-agent` が既定対象として優先される。

#### 効果
これにより、曖昧な依頼でも `everything-claude-code` 側へ流れにくくなり、主作業対象である `hermes-agent` をより自然に選べるようになった。
