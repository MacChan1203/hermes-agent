# Hermes Agent 開発ログ

## 基本情報
- プロジェクト名: Hermes Agent
- 開発者: Macちゃん
- 協力: ChatGPT
- 開発形態: 既存 `hermes-agent` のローカル改造・機能拡張
- 主な目的: 自然文で扱える、安全な CLI エージェントを作ること

---

## 開発目標
既存の Hermes Agent をベースに、以下を実現することを目標とした。

1. 自然文からコマンドを扱えるようにする  
2. 現在の作業ディレクトリを保持する  
3. 前回コマンドを再利用できるようにする  
4. terminal 実行結果を安定して返す  
5. 危険操作を防ぐ安全モードを導入する  
6. README を日本語化し、GitHub 公開まで完了する  

---

## 開発内容

### 1. 自然文コマンド変換の追加
`run_agent.py` に自然文をコマンドへ寄せる仕組みを追加した。

#### 追加した例
- `今どこ？` → `pwd`
- `フォルダ一覧見せて` → `ls`
- `親フォルダに戻って` → `cd ..`
- `gitの状態` → `git status`
- `gitの差分` → `git diff`
- `gitの履歴` → `git log --oneline -n 5`
- `前回のコマンドもう一回` → `repeat last command`

#### 設計意図
自然文から直接 LLM に解釈させるのではなく、Python 側で決定的にコマンドへ寄せることで、安定性と予測可能性を高めた。

---

### 2. CWD 状態保持の導入
現在の作業ディレクトリをセッションをまたいで維持できるようにした。

#### 実装内容
- `.hermes_cwd` に現在の CWD を保存
- 起動時に `.hermes_cwd` を読み込み
- `cd` は即時実行ではなく、内部状態変更として扱う

#### 効果
- `cd hermes-agent を実行して` のような操作後も状態を維持できる
- `pwd` や `ls` の結果が自然に連続するようになった

---

### 3. 前回コマンド再利用機能
前回実行したコマンドと、その実行時ディレクトリを記録する機能を追加した。

#### 実装内容
- `.hermes_last_command`
- `.hermes_last_command_cwd`

#### 対応した自然文
- `前回のコマンド`
- `前回のコマンドもう一回`

#### 効果
- 単に前回の文字列を記録するだけでなく、実行時ディレクトリも復元できるようになった
- CWD を変更したあとでも、前回の文脈で再実行可能になった

---

### 4. terminal 実行結果の整理
terminal ツールの結果を LLM に再解釈させず、そのまま返す fast-path を整備した。

#### 主な改善
- terminal の JSON 出力から `output` を直接返す
- `pwd` のときだけ `[cwd: ...]` 表示を付けない
- 通常コマンドは `[cwd: ...]` を付けて分かりやすくした

#### 効果
- コマンド結果が説明文に置き換わらず、そのまま返るようになった
- 実行結果の信頼性が向上した

---

### 5. Git リポジトリ外での親切エラー
`git diff` や `git log` を Git リポジトリ外で実行した場合、単なる Git エラーではなく案内つきで返すよう改善した。

#### 例
```text
⚠️ ここは Git リポジトリではありません
💡 例: cd hermes-agent を実行してから試してください
```

#### 効果
初心者にも分かりやすい UX になった。

---

### 6. 安全モードの導入
危険な操作や書き込み系操作を早期にブロックする仕組みを追加した。

#### 初期の課題
- `rm test.txt を実行して` がコマンドとして正しく抽出されず、LLM に流れて説明文が返ることがあった
- `git push を実行して` も同様に LLM に流れる場合があった

#### 解決策
`normalized_message = user_message.strip()` の直後に、危険意図を早期に判定する処理を追加した。

#### ブロック対象
- 削除系  
  - `rm`
  - 「削除」「消して」「消す」

- Git 書き込み系  
  - `git add`
  - `git commit`
  - `git push`
  - `git merge`
  - `git rebase`

- ファイル変更系  
  - `mv`
  - `cp`
  - `chmod`
  - `chown`
  - `sed -i`

- 外部取得・スクリプト実行系  
  - `curl`
  - `wget`
  - `bash`
  - `sh`

#### 最終的な返答例
```text
⚠️ 安全モードのため、削除系の操作は実行できません
💡 代わりに `ls` や `find` で対象を確認してください
```

```text
⚠️ 安全モードのため、Gitの書き込み操作は実行できません
💡 `git status`、`git diff`、`git log --oneline -n 5` は利用できます
```

#### 効果
- 危険操作が terminal に行かなくなった
- LLM にもフォールバックしなくなった
- 理由と代替案を案内できるようになった

## 発生した問題と解決

### 1. `extracted_cmd` の UnboundLocalError
#### 原因
`extracted_cmd` を定義する前に `if extracted_cmd:` を実行していた。

#### 対応
- コマンド抽出後に安全モード処理を置く
- さらに、危険意図の早期ブロックは `normalized_message` 直後に置く構成へ変更

---

### 2. `前回のコマンドもう一回` が誤って扱われる問題
#### 原因
`last command` 系との判定順が競合した。

#### 対応
- 再実行系を先に強制判定するように調整した

---

### 3. GitHub push 時の workflow 権限エラー
#### エラー内容
`.github/workflows/...` を含む commit を push しようとすると、PAT に `workflow` scope がないため拒否された。

#### 対応
- 問題の commit を `git reset --soft origin/main` でほどく
- workflow ファイルを staged から外す
- 必要なファイルだけ add し直して commit
- その後、push に成功

#### 学んだ点
- `git status` が clean でも、未pushコミットに不要ファイルが入っている場合がある
- `git diff --cached --name-only` で commit 対象確認が重要
